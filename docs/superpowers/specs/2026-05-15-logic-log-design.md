## 概述

`logic-log` 是一個 Claude Code plugin，用於自動捕捉並結構化對話中的邏輯思維與決策過程。

**核心定位：你的邏輯思維是主角，Claude 是結構化記錄員。**

每當對話中出現分析框架、解決方案論證、多維度考量時，自動整理成可攜帶的知識記錄——可即時查看、跨 session 引用、分享給他人。

---

## 使用情境

- 對話中提出分析框架或解決方案後，想保留完整推理過程
- 事後想回顧「我當時為什麼這樣決定」
- 想快速把自己的邏輯分享給其他人
- 在新的對話中引用過去某次的分析結論


## 觸發條件（SKILL.md 核心規則）

Claude 在以下情況必須輸出邏輯記錄區塊：

| 觸發情境 | 例子 |
|---------|------|
| 使用者提出分析框架 | 「我覺得這個問題要從 A、B、C 三個角度看…」 |
| 使用者說明為什麼某方案更好 | 「這樣做比 XX 好，因為…」 |
| 使用者列舉多維度考量 | 「我考量了成本、風險、擴展性…」 |
| 使用者做出方向性決定 | 確認某個技術/策略選擇 |
| Claude 提出解決方案 | 附上「為什麼適合你需求」的論證 |

## 儲存結構

```
~/.claude/logic-logs/
├── 2026-05-15.md       ← 當日完整記錄（含完整區塊）
├── index.md            ← 所有記錄的摘要索引，跨 session 引用用
└── export/
    └── 2026-05-15.md   ← 乾淨版，可直接分享
```

### 每筆 index.md 格式

```
#003 | 2026-05-15 14:23 | Skill+Hook架構 | 考量:依賴度·自動性·持久性 | 結論:零依賴+自動觸發
```

## 狀態列

顯示最近 3 筆，每筆包含主題 + 關鍵考量 + 結論：

```
💡 #003 Skill+Hook｜依賴度·自動性｜零依賴+自動觸發  ·  #002 Plugin格式｜可維護·可分享｜Markdown  ·  #001 記錄系統｜...
```

空間不足時，最新一筆顯示完整，舊的縮短為標題。

## /llog 指令（Claude Code 中使用）

| 指令 | 效果 |
|------|------|
| `/llog` | 顯示本次 session 所有記錄的摘要列表 |
| `/llog 3` | 顯示本 session 第 3 筆的完整內容 |
| `/llog today` | 今日所有記錄 |
| `/llog share` | 輸出乾淨格式，可直接複製分享 |
| `/llog search <關鍵字>` | 跨所有 session 搜尋 |
| `/llog #042` | 直接開啟第 042 筆（不限 session） |
| `/llog ref #042` | 把 #042 的邏輯注入當前對話 context，讓 Claude 繼續推理 |

---

## SessionStart Hook

每次開啟 session，自動把最近 5 筆記錄摘要注入給 Claude，讓 Claude 知道：
> 「上次你們決定了 X，理由是 Y，排除了 Z」

下次對話可直接引用，不用重新解釋背景。

## 技術決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| 儲存格式 | Markdown | 零依賴、人類可讀、可直接分享 |
| 架構 | Skill + Hook | 比純 Skill 可靠，比 MCP 輕量 |
| 搜尋 | `grep` / `ripgrep` | 無需資料庫，Markdown 本身就是索引 |
| 理論識別 | Claude 判斷 | 不需要外部 API，Claude 直接標注 |
